基于提供的源文档（例如，财务报告、分析报告、产品信息、技术解读），生成一个单一、完整、可直接运行的 HTML 文件。此文件必须动态地、可视化地呈现文档中的核心发现、关键数据和结构化信息，**同时保持内容的叙事性和逻辑完整性**，并严格遵守以下设计和实现要求：

**内容质量核心要求（重要）：**
1.  **保持叙事完整性：**
    *   **绝对禁止**只提取数据和要点而丢失上下文。必须保留源文档的叙事逻辑和内容贴合度。
    *   每个主要版块必须有**导语/背景说明**，解释这一部分讲的是什么、为什么重要。
    *   不要假设读者已经了解背景，每个关键概念都要有简要解释。
2.  **深入解读而非简单摘要：**
    *   **拓展解读内容**：对于技术概念、专业术语，提供通俗易懂的解释和实例。
    *   **补充关联信息**：当源文档提到某个概念时，可以适当拓展其意义、影响和应用场景。
    *   **连结上下文**：确保读者能理解各部分之间的逻辑关系和贡献。
3.  **完整的内容结构：**
    *   **开篇导读**：文章开头要有清晰的概述，说明这篇内容涵盖什么、为什么值得关注。
    *   **章节导言**：每个主要章节开头要有 2-3 句导言，简要说明这部分的核心讨论内容。
    *   **细节补充**：关键数据和结论要有充分的解释和上下文，不能孤立展示。
    *   **总结回顾**：文末要有综合性的整体回顾，串联所有要点。
4.  **合理的内容长度：**
    *   生成的可视化内容应该覆盖源文档的**所有重要章节**。
    *   不要为了追求简洁而牲牲内容完整性。可以有较长的页面，但要保证内容质量。
    *   每个要点卡片内可以有更多的解释性文字（不仅仅是数字）。
**核心要求：**
1.  **单一 HTML 文件输出：**
    *   最终交付物必须是**一个 .html 文件**，包含所有必要的 HTML 结构、CSS 样式（通过 `<style>` 标签或内联 Tailwind 类）和 JavaScript（通过 `<script>` 标签，包括 CDN 引入和初始化逻辑）。
    *   不允许使用外部 CSS 或 JS 文件，只允许在 HTML 文件内部使用 CDN 链接。
2.  **严格禁止使用图片：**
    *   **绝对禁止**在生成的 HTML 中包含任何 `<img>` 标签或图片链接（包括但不限于 Unsplash、Pexels、本地图片路径等）。
    *   使用 Font Awesome 图标、CSS 渐变、颜色块和数据可视化图表来实现视觉效果，而非依赖外部图片资源。
    *   如果需要视觉装饰，使用大号图标（Font Awesome）、emoji 符号或 CSS 样式来替代。
**视觉设计与布局：**
1.  **整体风格：**
    *   参考 Apple 官网及发布会风格——简洁、现代，具有清晰的信息层级。
2.  **布局核心：卡片式布局**
    *   **主卡片 (Main Cards)：** 用于主要版块（如执行摘要、财务状况、业务分部等）。使用大尺寸、风格明确的卡片样式。
    *   **迷你卡片 / 要点卡片 (Mini-Cards / Point-Cards) (关键布局元素)：**
        *   **目的：** 专门用于拆分和展示包含多个要点、风险、建议、特性、步骤或并行信息（例如，"关键发现"、"风险"、"建议"、"特性"）的段落或列表。
        *   **AI 任务：** 深入理解这些内容的语义。提取每个逻辑上独立、并列的核心要点。将每个要点转换为一个独立的迷你卡片。
        *   **布局：** 将这些迷你卡片排列在一个响应式网格中（例如，`grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 2xl:grid-cols-5 gap-4`）。在中大型屏幕上，目标是每行排列 3-5 个迷你卡片，以实现结构化、易于扫视的信息密度。
        *   **嵌套布局：** 主卡片可以包含迷你卡片网格。
3.  **长图优化布局规范（重要）：**
    *   **全局宽度控制：** 所有内容必须包裹在一个主容器中，使用 `max-w-6xl mx-auto px-6` 或类似类，确保内容宽度统一，避免部分内容过宽导致两侧留白不一致。
    *   **卡片等宽原则：** 
        *   **同层级卡片必须等宽**：同一个网格容器内的所有卡片（迷你卡片或主卡片）必须使用相同的 Grid 列数配置，确保每行卡片宽度一致。
        *   **禁止混合布局**：避免在同一视觉区域混用不同的网格配置（如一处用 3 列，另一处用 4 列），这会导致视觉不协调。
        *   **统一 Grid 配置**：推荐全局统一使用 `grid-cols-1 md:grid-cols-2 lg:grid-cols-3`（移动端 1 列，平板 2 列，桌面 3 列），确保所有卡片网格保持一致。
    *   **间距统一：** 所有卡片之间的间距（`gap`）必须统一，推荐使用 `gap-6` 或 `gap-8`，避免部分区域间距大、部分区域间距小。
    *   **内边距一致：** 同类型卡片的内边距（`padding`）必须一致，主卡片统一使用 `p-8` 或 `p-10`，迷你卡片统一使用 `p-6`。
    *   **垂直节奏：** 使用统一的垂直间距（`mb-12` 或 `mb-16`）分隔不同章节，营造良好的阅读节奏。
    *   **避免绝对定位**：除非必要（如固定背景效果），禁止使用 `absolute` 定位，优先使用 Flexbox 和 Grid 确保布局可控。
4.  **背景：**
    *   纯黑页面背景 (`#000000`)。
5.  **高亮颜色**
    *   **自动品牌色：** 尝试识别内容中主要品牌（例如，“小米”、“华为”）。如果成功，则使用其官方主 VI 色（例如，小米橙 `#FF6900`）作为**唯一的核心高亮色**。
    *   **备选方案：** 如果品牌/颜色识别失败，则使用专业的科技蓝 (`#00AEEF`) 或亮橙色 (`#FFA500`)。
    *   **应用：** 统一应用于关键文本（标题、核心数据）、大的强调数字、图标、图表元素、可选边框和渐变色。
6.  **科技感渐变 (Tech Gradient)：**
    *   **仅应用于高亮色：** 创建从 `rgba(高亮色, 0.7)` 到 `rgba(高亮色, 0.3)` 的透明度渐变。
    *   **用途：** 用作卡片/区域、图表区域或文本背景的微妙底色。**禁止使用多色渐变**。
7.  **卡片样式 (Card Styling)：**
    *   **背景：** 所有卡片（主卡片和迷你卡片）均使用深灰色背景（例如 `#1a1a1a` 或 `#222222`）。
    *   **圆角：** 主卡片使用较大圆角半径（`rounded-xl` 或 `rounded-2xl`），迷你卡片使用较小圆角半径（`rounded-lg`）。
    *   **分隔：** 使用细微边框（`border: 1px solid #333;`）或适合暗黑模式的轻微阴影（`shadow-md` 或 `shadow-lg`）。
    *   **内边距：** 确保所有卡片有足够的内边距（主卡片 `p-6` 或 `p-8`，迷你卡片 `p-4` 或 `p-6`），防止内容贴边或重叠。
8.  **主标题强化 (Main Title Enhancement)：**
    *   使主中文标题显著增大（例如，`text-5xl` 或 `text-6xl`, `font-bold`）。
    *   在其下方添加一个较小的、对应的英文标题（例如，`text-xl` 或 `text-lg`, `font-semibold`，颜色稍浅，如 `text-gray-300` 或 `text-gray-400`）。示例：“Xiaomi Corporation 2024 Annual Financial Report Analysis”。
**内容呈现与布局（核心优化）：**
1.  **全面的基础数据：**
    *   准确提取并展示源文档中的所有关键信息、核心数据点（特别是财务数据、增长率、市场份额）、结论和分析。
2.  **核心要点提取与卡片化（关键）：**
    *   **语义理解：** 超越简单的句子分割。理解逻辑和语义，识别段落或列表**内部独立、并列的核心观点/元素**。
    *   **转换：** 将每个提取出的核心要点转化为**一个独立的迷你卡片**。
    *   **目标：** 将复杂信息解构成结构化、可视化的网格，使其成为易于扫视的单元。
3.  **迷你卡片内部结构与细节（关键）：**
    *   **结构优先级：**
        *   **强调数字优先：** 如果核心要点包含一个关键、突出的数字/指标，则将该**数字/指标本身**作为顶部元素，使用超大、粗体字号（例如 `text-5xl` 或 `text-6xl`, `font-bold`，使用高亮色）。目的是为了最大化视觉冲击力。
        *   **文本标题：** 如果核心要点是概念性的或数字是次要的，则使用一个**简洁、加粗的中文短语**（理想情况 3-5 个汉字）作为顶部元素，使用大字号（例如 `text-3xl` 或 `text-4xl`, `font-bold`，白色或高亮色）。
    *   **支撑文本（内容深度强化）：** 在大的数字/标题下方，使用较小字号（例如 `text-sm`, `text-gray-400`）。
        *   **当大元素是数字时：** 不仅解释该数字代表什么，还要说明**为什么这个数字重要**、**它的含义是什么**、**它与整体内容的关联**。提供 2-4 句说明文字。
        *   **当大元素是文本标题时：** 提供具体细节、解释、数据支撑或影响。**至少 3-5 句话**来充分解释这个要点。
    *   **可选双语副标题（选择性应用）：**
        *   在适合增强设计感的地方（尤其是在大的中文数字或文本标题下方），添加一个**非常简洁的英文短语**，使用小字号（例如 `text-xs` 或 `text-sm`）、常规字重和柔和的颜色（例如 `text-gray-500` 或 `text-gray-400`）。示例：“YoY Growth”，“Total Revenue”，“Market Share”。**审慎地应用**以增加视觉风格，而非死板地用于每个卡片。
    *   **内容深度与简洁的平衡：** 确保每个卡片聚焦一个核心要点，但该要点的解释必须**充分且有价值**。避免空洞的数字展示，读者应能理解每个要点的来龙去脉。
4.  **强烈的视觉层级：**
    *   利用显著的**字号差异**（例如，数字用 `text-5xl/6xl` vs 文本标题用 `text-3xl/4xl` vs 支撑文本用 `text-sm` vs 英文副标题用 `text-xs`）、**字重**（`font-bold` vs `font-normal`）和**颜色**（高亮色、白色、灰色系）来在主要信息（大数字/标题）和次要信息（支撑文本、英文副标题）之间创建清晰的视觉区分。
5.  **语言策略：**
    *   **主要语言：** 使用**中文或数字**承载核心信息、主标题（大的中文部分）和迷你卡片的大元素。确保它们在视觉上占主导地位（大字号、加粗）。
    *   **次要/装饰性语言：** 使用**英文**作为主标题的副标题以及可选的、小的迷你卡片副标题。对这些元素使用较小字号和较低的强调度。如果源文档中存在英文术语，为确保技术准确性应予保留。
**图形元素与图表：**
1.  **图标 (Font Awesome)：**
    *   **来源：** 通过 CDN 引入 Font Awesome (v5/v6)。
    *   **风格：** 偏好简洁、现代的**线框风格 (outline-style)** 图标。
    *   **使用：** 放置于主标题附近，可选择性地（且需微妙地）用于迷你卡片内部（靠近标题处）、列表前缀等。**严格禁止使用 Emoji 作为功能性图标**。颜色应协调；关键图标可使用高亮色。
    *   **布局约束（关键）：** 
        *   图标与文字必须使用 flexbox 布局（`flex items-center gap-2` 或 `gap-3`），确保有足够间距。
        *   避免使用绝对定位（`absolute`）放置图标，除非有明确的容器约束。
        *   图标大小要适中（`text-2xl` 到 `text-4xl`），避免过大导致布局错乱。
        *   如果图标和文字在同一行，使用 `inline-flex` 或 `flex` 容器包裹，设置 `items-center` 垂直居中。
        *   **严格避免图标与文字重叠**：始终使用 `gap-*` 类或 `margin` 来保持间距。
2.  **数据可视化 (推荐 Chart.js)：**
    *   **应用场景：** 用于展示趋势、增长率、构成（饼图/环形图）、比较（柱状图）等适合的数据 [引用：数据可视化最佳实践]。
    *   **技术：** 通过 CDN 嵌入 Chart.js。
    *   **位置：** 放置在讨论财务或业务分析的相关主卡片内部。
    *   **样式：** 图表颜色必须与暗黑主题和高亮色保持一致。确保图表清晰、易读且响应式。
    *   **容器高度限制（关键）：** 所有包含 `<canvas>` 的图表容器必须设置明确的高度。使用 `style="height: 400px"` 或 Tailwind 类如 `h-96`（384px）、`h-80`（320px）。推荐高度范围：300-500px。这对于防止 Chart.js 渲染异常至关重要。
    *   **容器约束（关键）：** 
        *   **必须**将每个 `<canvas>` 元素包裹在一个具有明确高度限制的容器 `<div>` 中。
        *   容器必须设置固定高度（例如 `h-64`、`h-80`、`h-96`）或最大高度（例如 `max-h-96`）。
        *   在 Chart.js 配置中，必须设置 `options.maintainAspectRatio = false` 和 `options.responsive = true`。
        *   示例结构：`<div class="h-80"><canvas id="myChart"></canvas></div>`
**技术与动画：**
1.  **技术栈：**
    *   HTML5, TailwindCSS 3+ (**使用 script 标签加载 CDN**：`<script src="https://cdn.tailwindcss.com"></script>`), 原生 JavaScript (用于 Intersection Observer/图表初始化), Font Awesome (**使用本地文件**：`/css/vendor/fontawesome/all.min.css`), Chart.js (**使用本地文件**：`/js/vendor/chart.umd.min.js`)。
    *   **资源加载优化**：TailwindCSS 必须使用 `<script>` 标签而非 `<link>` 标签加载；Font Awesome 和 Chart.js 使用本地文件确保快速加载。
    *   **布局最佳实践：** 优先使用 Flexbox（`flex`）和 Grid（`grid`）进行布局，避免使用 `float` 或复杂的绝对定位。确保所有元素有明确的容器和间距控制。
2.  **动画 (CSS Transitions & Intersection Observer)：**
    *   **触发：** 当元素（所有主卡片、所有迷你卡片、其他内容块）滚动进入视口时。
    *   **效果：** 平滑、微妙的**淡入/向上滑动**效果（模仿 Apple 风格）。通过 JavaScript 的 `Intersection Observer API` 添加/移除 CSS 类来触发 `CSS Transitions` 实现。确保动画性能流畅。为网格项应用轻微延迟以产生交错效果。
3.  **响应式设计：**
    *   **强制要求**。使用 Tailwind 的响应式修饰符（特别是针对网格布局），确保在手机、平板和桌面设备上均具有出色的显示效果和可用性。
**最终输出：**
*   生成一个**单一、可运行的 .html 文件**，该文件精确实现了上述所有要求，特别注意：
    *   **内容完整性**：保持叙事逻辑，提供充分的背景和解释
    *   **视觉化呈现**：使用卡片布局展示核心要点
    *   **深度解读**：每个要点都有充分的解释和上下文
    *   **美学一致性**：整体风格统一，使用高亮色和暗黑主题
